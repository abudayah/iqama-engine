refactor: remove Friday Block mechanism for simplicity and accuracy

BREAKING CHANGE: Prayer times now use each day's own astronomical times
instead of locking to Friday's values for the week.

Changes:
- Remove Friday Block logic from cache.service.ts
- Update rules.service.ts to not accept fridayRaw parameter
- Delete friday-block.rule.ts (no longer needed)
- Remove Friday Block references from rule comments
- Update tests to verify daily astronomical accuracy
- Update documentation (README.md, steering files)

Rationale:
- Simpler codebase with less complexity
- More accurate - each day uses its actual astronomical time
- Easier to understand and maintain
- Prayer time changes are small (5-10 minutes over a week)

Impact:
- Fajr: Changes 2-3 times per week (e.g., 05:35 → 05:30 → 05:25)
- Asr: Minimal changes due to rounding (usually stays same)
- Isha: Changes 2-3 times per week (e.g., 22:25 → 22:30 → 22:35)
- Maghrib: Already changed daily (not affected by Friday Block)
- Dhuhr: Already fixed by DST (not affected by Friday Block)

Tests: All unit and e2e tests passing (11 tests total)
